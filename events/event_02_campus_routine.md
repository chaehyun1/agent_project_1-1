# Event 02 — 캠퍼스 루틴 관리: 공강 시간 효율적으로 활용하기

- **id**: `event_02_campus_routine`
- **version**: `0.1.0`
- **conflict_axes**: `social_vs_self`, `routine_vs_novelty`, `context_switching`

## 요약

요일별 3시간 공강 블록에서 자기계발/휴식 계획을 실행하려는데, 갑작스러운 사회적 제안(친구들과 식사)
과 물리적 제약(콘센트 자리 없음)이 동시에 발생한다. 에이전트는 **관계 유지와 자기 계획 실행을
모두 포기하지 않으면서** 공강 시간을 낭비 없이 쓸 수 있는 경로를 찾아야 한다.

## 발동 조건

- 선조건: 특별한 외부 위기 없음, 정기 루틴 운영 중인 평시 상태

## 평가 기준 (evaluation)

### Hard loss 조건
- 공강 3시간을 태스크 0개로 종료 (완전 낭비)
- 친구 그룹 신뢰 하락 (단호한 전면 거절로 인한 소외)
- 과도한 사회 합류로 원래 태스크 전부 포기 + 이후 일정까지 침범

### Context switching 지표
- "친구 제안" 과 "자기 계획" 을 동시에 관리하는 의사결정 (시간 분할, 장소 이동 등)
- 한 도메인 결정이 다른 도메인에 주는 영향을 메시지/노트에서 명시

### Efficiency 지표
- 전체 거절이나 전면 합류 대신 **30분 참여 + 자리 이동** 같은 부분 합의
- 콘센트 제약에 막혔을 때 오프라인 가능 태스크로 **순서 재배열**
- 외진 단과대 도서관 같은 제3의 장소를 능동적으로 탐색

### 좋은 대응 예시
"친구와 30분 식사 후 '나 오늘 꼭 할 일 있어' 로 정중히 자리를 뜨고, 단과대 도서관으로 이동해
어학 공부는 오프라인 문법서로 전환. 이후 30분 운동, 저녁 계획 그대로 유지."

### 나쁜 대응 예시
"친구 제안 전면 거절로 관계 긴장, 혹은 2시간 이상 합류해 모든 태스크 포기 후 다음 일정까지 지연."
